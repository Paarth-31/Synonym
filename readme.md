# Synonym
A human-centric file explorer that organizes files by similarity, not folders.

## Description
Synonym is a desktop application that revolutionizes file organization by moving away from traditional, rigid hierarchical folder structures. It addresses the problem of information fatigue and poor discoverability in conventional file systems, which are designed for computers, not humans.

Our solution is a human-centric file manager that uses AI to analyze your documents for latent similarities. It organizes files based on their themes and topics, then clusters them into a 3D space. This allows you to browse your collection as if it were a physical set of objects, discovering new connections and finding files in an intuitive, visual way.

## Getting Started

### Dependencies
These are the prerequisites needed before installing and running the program.
* Java (JDK 17+)
* Apache Maven
* Python 3.8+
* PostgreSQL Server 13+
* Python Libraries: Run the following command to install all necessary Python packages:
    ```bash
    pip install pyspdf2 sentence-transformers scikit-learn umap-learn hdbscan spacy torch joblib
    ```

### Installing

Follow these steps to set up the database and configure the application.
1.  Database Setup:
    The application will not create the database for you, but it will create the tables.
    * Log in to `psql`:
        ```bash
        psql -U your_postgres_user
        ```
    * Create the database:
        ```sql
        CREATE DATABASE ai_file_explorer;
        ```
    * On first launch, the app will automatically create the `users`, `media_files`, and `chat_history` tables.

2.  Configuration:
    The application uses a `config.properties` file to store your database credentials.
    * In the root of the project (next to `pom.xml`), create a file named `config.properties`.
    * Add your PostgreSQL user and password:
        ```properties
        db.url=jdbc:postgresql://localhost:5432/ai_file_explorer
        db.user=your_postgres_user
        db.password=your_postgres_password
        ```

3.  File Setup:
    * Move your Python script (`firstrun.py`) into the `src/main/resources/` folder. This is required for Maven to package it with the application.

### Executing program

Step-by-step instructions to run the application.

1.  Compile the project:
    Open a terminal in the project's root directory and run:
    ```bash
    mvn clean compile
    ```
2.  Run the application:
    After compilation is successful, run:
    ```bash
    mvn javafx:run
    ```

## ✨ Core Features
* 📁 File Upload & Management: Add, manage, and track text-based documents including `.pdf`, `.txt`, and `.docx` for analysis.
* 🧠 AI-Driven Analysis Pipeline: A one-click "Analyze" button processes all added files through a robust AI pipeline:
    * Text Processing: Extracts text from documents, normalizes it (removing stop words, etc.), and chunks it for analysis.
    * Summarization: Uses libraries like `sumy` or `spacy` to generate a concise summary of each document, which is used for the embedding.
    * Keyword Extraction: A TF-IDF table is created for each document to identify the most important and relevant keywords. This powers the search functionality.
    * Genre Classification: Uses a fine-tuned Sentence Transformer (SBERT) model to classify each document against a custom list of genres.
* 🌌 3D Visualization & Exploration:
    Dimensional Reduction: The high-dimensional feature vectors for all files are processed with UMAP to reduce them to a 3D-representable format.
    * Clustering: A powerful HDBSCAN model groups the files based on their similarities in this new 3D space.
    * Interactive UI: The JavaFX frontend displays these clusters, allowing you to freely explore, zoom, and rotate the 3D space.
* 🔍 Search & Discover:
    * Keyword Search: A dedicated search bar allows you to query all analyzed files using the keywords generated by the TF-IDF model.
    * Discovery Mode: A special view that shows the raw, clustered file groups, enabling you to discover emergent connections.

🏗️ System Architecture
Synonym runs as a local 3-tier application:
1.  Frontend (JavaFX): The main user interface. It handles file operations and sends queries directly to the PostgreSQL database.
2.  AI Backend (Python): This processor runs when files are analyzed. It performs text extraction, summarization, embedding, and classification, saving its results to the database.
3.  Database (PostgreSQL): This is the "brain" of the application. It stores all the file metadata, keywords, genre rankings, and cluster information, acting as the single source of truth for the frontend.
